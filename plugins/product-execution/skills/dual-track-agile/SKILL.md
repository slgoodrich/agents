# Dual-Track Agile

## Overview
Development approach that runs discovery and delivery in parallel, ensuring teams build the right thing while maintaining delivery velocity.

## Two Tracks

### Discovery Track
**Purpose**: Learn what to build
- Customer research
- Problem validation
- Solution exploration
- Prototyping
- Assumption testing

### Delivery Track
**Purpose**: Build and ship
- Development
- Testing
- Deployment
- Monitoring
- Iteration

## How Tracks Work Together

### Flow
1. **Discovery** identifies opportunities
2. Validated opportunities → **Delivery backlog**
3. **Delivery** builds and ships
4. **Discovery** learns from shipped features
5. Cycle continues

### Cadence
- Discovery runs 1-2 sprints ahead
- Continuous feedback loop
- Regular synchronization points

## Team Structure

### Product Trio
- **Product Manager** - Prioritization, strategy
- **Designer** - User experience, interaction
- **Engineer** - Technical feasibility

### Whole Team Involvement
- Engineers in discovery (feasibility)
- Designers in delivery (refinement)
- PM in both (orchestration)

## Discovery Activities
- Customer interviews
- Usability testing
- Prototype testing
- Spike solutions
- Assumption mapping
- Opportunity assessment

## Delivery Activities
- Sprint planning
- Development
- Code review
- Testing
- Deployment
- Retrospectives

## Benefits
- Reduced waste
- Better product-market fit
- Validated features
- Continuous learning
- Balanced pace

## Anti-patterns
- Discovery too far ahead (ivory tower)
- No discovery (feature factory)
- Separate discovery team (handoffs)
- Only doing discovery (analysis paralysis)

## When to Use
Product teams, continuous discovery, agile environments, customer-centric development
