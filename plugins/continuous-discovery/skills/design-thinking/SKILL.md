# Design Thinking

## Overview
Human-centered approach to innovation that integrates customer needs, technology possibilities, and business requirements.

## Five Phases

### 1. Empathize
**Goal**: Understand users deeply
- User interviews
- Observation
- Immersion
- Empathy maps
- Journey mapping

### 2. Define
**Goal**: Frame the problem
- Point of view statements
- Problem statements
- How Might We questions
- User needs insights

### 3. Ideate
**Goal**: Generate solutions
- Brainstorming
- Worst possible idea
- SCAMPER technique
- Mind mapping
- Quantity over quality

### 4. Prototype
**Goal**: Make ideas tangible
- Low-fidelity prototypes
- Paper prototypes
- Interactive mockups
- Role-playing
- Storyboards

### 5. Test
**Goal**: Learn and iterate
- User testing
- Feedback collection
- Refinement
- Iteration

## Empathy Map
**Says** - What user says out loud
**Thinks** - What user might be thinking
**Does** - Actions and behaviors
**Feels** - Emotional state

## How Might We (HMW) Questions
Transform problems into opportunities
- Start with "How might we..."
- Suggest possibilities
- Generate options
- Inspire ideation

## Ideation Techniques
1. **Brainstorming** - Free-flowing ideas
2. **Brainwriting** - Silent idea generation
3. **Worst Possible Idea** - Reverse thinking
4. **SCAMPER** - Substitute, Combine, Adapt, Modify, Put to other uses, Eliminate, Reverse
5. **Crazy 8s** - 8 ideas in 8 minutes

## When to Use
Innovation challenges, user experience design, complex problem-solving, cross-functional collaboration
