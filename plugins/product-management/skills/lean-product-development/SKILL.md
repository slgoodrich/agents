# Lean Product Development

## Overview
Methodology for building products through validated learning, rapid experimentation, and iterative development.

## Build-Measure-Learn Loop

### Build
Create minimum viable product (MVP)
- Simplest version to test hypothesis
- Focus on learning, not perfection

### Measure
Collect actionable metrics
- Quantitative data
- Qualitative feedback
- Leading indicators

### Learn
Make data-driven decisions
- Pivot or persevere
- Iterate based on insights

## MVP Types
1. **Concierge MVP** - Manual service delivery
2. **Wizard of Oz MVP** - Automated appearance, manual backend
3. **Landing Page MVP** - Gauge interest before building
4. **Prototype MVP** - Interactive mockup
5. **Single Feature MVP** - One core capability

## Validated Learning
- Start with hypothesis
- Define success criteria
- Design minimum experiment
- Measure real behavior
- Make informed decision

## Pivot Types
1. **Zoom-in pivot** - Single feature becomes whole product
2. **Zoom-out pivot** - Whole product becomes single feature
3. **Customer segment pivot** - Different target customer
4. **Customer need pivot** - Different problem to solve
5. **Platform pivot** - App to platform or vice versa
6. **Value capture pivot** - Different monetization
7. **Engine of growth pivot** - Different growth strategy

## Innovation Accounting
1. **Baseline** - Current state metrics
2. **Tuning** - Optimize toward ideal
3. **Pivot decision** - Change strategy if not working

## When to Use
Startup validation, new feature testing, market uncertainty, resource constraints
