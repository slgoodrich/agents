# Story Mapping

## Overview
Visual technique for organizing user stories into a useful model to understand system functionality, prioritize features, and plan releases.

## Story Map Structure

### Backbone (Top Row)
**User Activities** - High-level user goals
- Major steps in user journey
- Left to right flow
- Typically 5-10 activities

### Walking Skeleton (Second Row)
**User Tasks** - Actions to complete activities
- Specific tasks within each activity
- Organized chronologically
- Foundation for stories

### User Stories (Below Tasks)
**Details and Alternatives** - Specific implementations
- Ordered by priority (top to bottom)
- Multiple ways to accomplish task
- Increasing sophistication

## Release Planning with Story Maps

### Horizontal Slicing
**Release 1** - Minimum viable journey
**Release 2** - Enhanced experience
**Release 3** - Delightful experience

### Criteria
- Walking skeleton first
- Must complete full journey
- Add depth in later releases

## Building a Story Map

### Steps
1. **Frame the problem** - Who and why
2. **Map the big picture** - User activities
3. **Explore** - Tasks and details
4. **Slice out releases** - Prioritize vertically
5. **Slice out a learning strategy** - Validate assumptions

### Map Elements
- **Activities** (swim, bike, run)
- **Tasks** (enter water, start swimming)
- **Stories** (adjust goggles, breathe)

## Benefits
- Shared understanding
- Identify gaps
- Prioritize effectively
- See big picture and details
- Plan incremental delivery

## When to Use
Release planning, backlog organization, MVP definition, team alignment, complex feature planning
