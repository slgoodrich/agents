- **Hybrid Performance**: Fast Haiku tools (20x cheaper) + strategic Sonnet workflows + expert Sonnet agents

## Hybrid Model Strategy

This toolkit uses a strategic three-tier model approach for optimal performance and cost efficiency across different PM tasks.

### Model Selection Philosophy

**Right model for the right task:**
- **Fast analysis** → Haiku (framework tools, structured analysis)
- **Complex orchestration** → Sonnet (multi-week workflows, strategic planning)
- **Deep expertise** → Sonnet (expert agents, nuanced decisions)

This approach delivers **30-40x cost efficiency** on routine tasks while maintaining expert-level quality for strategic work.

### Choosing the Right Tier

**Decision tree:**

```
Is it a well-defined framework? (SWOT, ROI, Business Model Canvas)
  ↓ YES → Use Haiku Tool (Tier 1)
  ↓ NO  → Continue

Is it a multi-week process? (Planning, Launch, Discovery Sprint)
  ↓ YES → Use Sonnet Workflow (Tier 2)
  ↓ NO  → Continue

Do you need expert consultation? (Strategy, Research, Prioritization)
  ↓ YES → Use Sonnet Agent (Tier 3)
```

**Combining tiers:**
```bash
# 1. Quick analysis with Haiku tool
/competitive-intelligence:swot-analysis "Expand to enterprise market"

# 2. Deep strategic thinking with Sonnet agent
Tell product-strategist: "Based on this SWOT, create expansion strategy"

# 3. Execute with Sonnet workflow
/competitive-intelligence:market-entry "Enterprise market expansion"
```

---

### Model Override

All tools and workflows support the `--model` flag for flexibility:

```bash
# Force Sonnet for deeper analysis
/competitive-intelligence:swot-analysis "Complex competitive landscape" --model sonnet

# Force Haiku for faster workflow (use cautiously)
/strategic-planning:quarterly-planning "Q2 2025" --model haiku
```

**When to override:**
- **Haiku → Sonnet**: Complex edge cases, strategic importance, need more nuance
- **Sonnet → Haiku**: Time-critical, budget constraints, simple scenario

## Best Practices

### Choosing the Right Tool

1. **Use Commands for Speed**: Commands are faster for straightforward, template-based tasks
   - Example: `/requirements-engineering:prd "feature name"` (30 seconds)
   - Best for: Quick document generation, structured outputs

2. **Use Haiku Tools for Analysis**: Framework tools are fast and cost-efficient
   - Example: `/competitive-intelligence:swot-analysis "strategic decision"` (30 seconds)
   - Best for: Structured analysis, decision frameworks, routine calculations

3. **Use Sonnet Agents for Complexity**: Agents provide deep expertise and nuanced guidance
   - Example: `Tell product-strategist: "complex strategic question"`
   - Best for: Complex decisions, strategic thinking, interactive refinement

4. **Use Sonnet Workflows for Processes**: Multi-step workflows orchestrate complete initiatives
   - Example: `/strategic-planning:quarterly-planning "Q2 2025"`
   - Best for: Multi-week processes, coordinated deliverables, strategic initiatives

### Effective Usage Patterns

5. **Combine Tiers**: Use Haiku tools → Sonnet agents → Sonnet workflows for complete coverage
   ```bash
   /competitive-intelligence:swot-analysis "market expansion"  # Fast analysis
   Tell product-strategist: "Based on SWOT, create strategy"  # Deep thinking
   /competitive-intelligence:market-entry "Execute strategy"  # Complete process
   ```

6. **Reference Skills**: Agents automatically load PM frameworks when you mention them
   - Mention "RICE" to activate prioritization frameworks
   - Mention "Jobs-to-be-Done" to activate JTBD methodology
   - Mention "AARRR" to activate pirate metrics

7. **Iterate and Refine**: Generate drafts quickly, then refine through conversation
   ```bash
   /requirements-engineering:prd "feature"  # Fast draft
   Tell requirements-engineer: "Add edge cases for enterprise customers"  # Refine
   Tell technical-pm: "Review technical feasibility"  # Validate
   ```

8. **Provide Context**: The more context you provide, the better the output
   - Company stage (startup, scale-up, enterprise)
   - Product type (B2B SaaS, consumer, marketplace)
   - Current metrics and goals
   - Team size and constraints

## FAQ

**Q: Can I use this without product management experience?**
A: Yes! The agents teach PM concepts while helping you. Start with `product-strategist` for guidance.

**Q: Does this replace PM tools like Jira or Linear?**
A: No, this complements them. Use this for thinking and planning, then implement in your PM tools.

**Q: Can I customize agents or commands?**
A: Yes! Fork the repo and modify to your needs. Contributions back are welcome.

**Q: What's the difference between agents and commands?**
A: Agents are interactive experts you converse with. Commands are quick utilities for specific tasks.

**Q: Do I need all 18 plugins?**
A: No! Install only what you need. Most PMs use 5-10 plugins regularly.

**Q: Can I use this for non-software products?**
A: The frameworks are universal, but examples are software-focused. Adapt as needed.

**Q: How does the hybrid model strategy work?**
A: Haiku tools handle fast framework analysis (20x cheaper), Sonnet agents provide expert consultation, and Sonnet workflows orchestrate multi-week processes. This balances speed, cost, and quality.

**Q: What's the token-efficient design?**
A: Each plugin loads only its agent and related commands. Installing 5 plugins loads far less context than a monolithic system, improving response speed and reducing costs.
